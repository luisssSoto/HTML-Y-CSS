Ordenar un CSS

Problemas comunes que pueden surgir al no ordenar un CSS

    Errores comunes
  • Reglas no utilizadas
  • Reglas duplicadas, que indican lo mismo o son contradictorias
  • Propiedades heredadas que pasan desapercibidas
  • El tiempo que se pierde para localizar una regla entre miles de lineas


Consejos para ordenar un CSS

Consejos
Organizar las reglas por orden alfabetico
a, div, em, .importante, #principal, table
Organizar las reglas por orden de aparición
html, body, h1, h2, div, #principal, p, .importante
Separar las reglas según la función que realizan:
por un lado las reglas que modifican los bordes, por otro lado las reglas que modifican los colores, por otro lado las reglas que definen la maquetación
Organizar las propiedades de una regla por orden alfabético
border, font-size, margin, padding, text-align
Organizar las propiedades de una regla del exterior del elemento al interior:
margin, border, padding, text-align, font-size

Facilitar la lectura de CSS

    Facilitar la lectura
  1. Espacios en blanco en el CSS, es decir para cada regla dejar los suficientes espacios y no todo en una sola linea
  2. Comentarios: se pueden hacer comentarios para describir reglas etc con la siguiente sintaxis /*Comentario*/
  3. Agrupación de selectores: Si una regla se repite en varios selectores estos se pueden agrupar separando cada selector mediante una coma y espacio: h1, h2
